Tabelas

Através da classe table as tabelas recebem o estilo para as tags th e td.

Caso botões de ação sejam necessários nas tabelas, os mesmos devem ser inseridos na última coluna.

Mais informações em: http://twitter.github.com/bootstrap/base-css.html#tables

Marcação

Tag Descrição
<table> Container principal que representa a tabela
<thead> Elemento que contêm as linhas de cabeçalho da tabela
<tbody> Elemento que contêm as linhas com os dados da tabela
<tr> Elemento que contêm as celulas da tabela
<td> Celula
<th> Celula especial para a descrever a coluna de uma linha.
<caption> Descrição da informação que a tabela contêm. Essa informação é útil para leitores de tela, onde descreve ao usuário com uma entonação direfente, qual é o conteúdo que será lido em seguida, como uma introdução.

Opções

Nome Classe CSS Descrição
Padrão Nenhuma Sem estilos, apenas colunas e linhas
Basica .table Apenas linhas horizontais para separar o conteúdo
Com borda .table-bordered Bordas arredondadas e borda para fechar a tabela
Zebra .table-striped Intercala as cores de fundo das linhas entre branco e cinza claro para acilitar a visualização
Condensada .table-condensed Corta o espaçamento da tabela pela metade em todas as celulas.
Ótimo para tabelas com muito conteúdo que devem ficar acima da barra de rolagem e economizar espaço.

Aplicação da Guideline

Estilo de tabela padrão

As tabelas nas aplicações da RNP devem receber as classes .table , .table-bordered e a classe
.table-striped deve ser adicionada apenas quando a tabela possuir uma massa grande de informações. Caso contrário seu uso não é necessário para evitar a poluição visual.

Seguir o exemplo ao lado para Nomes (Textos), a esquerda, Datas e Números centralizados, e Valores a direita.

Para responder a uma heuristica sobre uso de elementos visuais, será utilizado link para edição nas tabelas. Como pode ver no exemplo ao lado na coluna NOME. Tais links serão no nome principal do objeto, como no caso de perfil de usuário, sera no nome do usuário. A localização da coluna principal de edição será decidido pelos profissionais de UX e RNP.

Heurísticas do uso de tabelas

Cabeçalhos

Todo o cabeçalho das tabelas por padrão estarão em caixa alta, indiferente do texto inserido estar ou não, isso já é um padrão aplicado (implementado no CSS).

Explique do que a tabela se trata

Dê a tabela um título descritivo. Mostre o título da tabela acima do header, alinhado à esquerda.

Reduza e comprima as informações

Mostre apenas as informações que o usuário realmente necessita, mas coloque formas de exergar os detalhes. Overlays, inlays e tooltips ajudam o usuário a ficar na página e manter a fluidez da navegação Resuma as informações sempre que necessário.

O nível de precisão dos dados numéricos devem ter uma precisão de acordo com as necessidades do usuário. Quanto menos decimais, mais rápido o usuário lê e assimila a informação.

Tabelas com muitos dados

Quando a quantidade de informação disposta na tela for grande e complexa usar:

  • Paginação
  • Sorting
  • Busca

Campo de busca

Utilizar o campo de busca sempre que houver paginação na tabela.

Não use elementos visuais excessivamente

Sempre que possível evite o uso excessivo de elementos visuais. Os dados da tabela devem ter mais destaque visual que sua estrutura. Use cores leves, bordas finas. Evite sempre que possível o uso de uma coluna de "ações" com ícones e links (Ex.: Ver detalhes, editar, remover). O acesso ao detalhe das informações pode ser acessado ao clicar em uma coluna já existente (Ex.: Nome do usuário).

Destaque a informação mais importante

Usando algum estilo diferente (link, negrito, cores ou sublinhado) devemos destacar visualmente a informação mais relevante entre todas as colunas. Evite o uso excessivo de destaques. Ao enfatizar muitas coisas você não acaba enfatizando nada.

Alinhamento

  • Textos: Alinhas a esquerda.
    Utilize a classe .text-left
  • Números: Alinhar à direita, para facilitar a comparação e soma dos valores.
    Utilize a classe .text-right
  • Datas, horas e números de ordenação: Alinhar ao centro.
    Utilize a classe .text-center

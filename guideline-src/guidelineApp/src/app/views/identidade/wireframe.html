Wireframe

Introdução

Os wireframes são desenvolvidos utilizando um grid dividido em 12 colunas. Todo o layout é adicionado a um container que possui um tamanho fixo de 1170px para desktops, 724px para tablets e para dispositivos menores esse tamanho se torna dinâmico. Essa medida é ajustada de acordo com o dispositivo acessado.
As Grids são muito úteis e funcionais e aceleram o processo de produção. As páginas web essencialmente giram em torno de formas de caixas. Inevitavelmente, estes retângulos têm de se unir de alguma forma ou de outra para formar um desenho. Um sistema de grid é uma técnica adotada para organizar conteúdo a muito tempo em páginas impressas e nos últimos anos vêem sendo utilizada em peso também para aplicações web.

http://getbootstrap.com/css/#grid

Para o desenvolvimento dos wireframes, considerar a fonte Helvetica Neue ou Helvetica, tamanho 143x.

O Twitter Bootstrap já possui um sistema de grids incorporado na sua biblioteca.

Classes de Uso

  • col-xs-*
  • col-sm-*
  • col-md-*
  • col-lg-*
  • col-xs-offset-*
  • col-sm-offset-*
  • col-md-offset-*
  • col-lg-offset-*

Onde * pode ser substituído pelos números de 1 a 12.

Nesting:

O sistema de grid possibilita que as colunas .col-*-[1-12] possam ser alinhadas uma dentro das outras. Isso torna flexível e simples a criação de layouts complexos.

O que não fazer

  • Caso o conteúdo não preencha completamente a tela, alinhá-lo sempre à esquerda. Nunca à direita.
  • Não permitir que o conteúdo encoste nas margens do Browser.
  • Não utilizar cores que não estejam definidas pela Guideline.
  • Não utilizar ícones que não estejam na Guideline.

Construindo o projeto

Pré-requisitos
  1. Node
  2. Gulp
  3. Bower

Siga o processo de instalacão utilizando os links acima na ordem em que aparecem na lista.

Estrutura de Pastas

Raíz do Projeto

  • /src

    Projeto Guideline

    Nesta pasta ficam as bibliotecas e arquivos para a criação de customização dos componentes (.js, .scss).
    A versão da Guideline trabalha com o sistema de versionamento do Bower, sendo assim, ao informar a version no arquivo bower.json, você automaticamente gera uma nova versão da guideline.

    Para a inicialização do comando gulp build é necessário a instalação dos módulos do Node. Para realizar esta ação, execute o comando npm install, e uma pasta chamada node_modules será criada.

    Para instalar as bibliotecas de componentes da Guideline RNP, execute o comando bower install, e uma pasta chamada bower_components será criada.

    A partir deste ponto, a aplicação estará apta para realizar a geração de build do projeto Guideline RNP.
    O Gulp roda o comando no arquivo padrão gulpfile.js.

    Nota Ao gerar o comando gulp build, a pasta raíz chamada /release e a pasta /guidelineApp/src/release serão atualizadas com os arquivos gerados pela build.

  • /guidelineApp

    Documentação Guideline

    Nesta pasta ficam os arquivos da Aplicação Web da Documentação da Guideline RNP.
    A Documentação do projeto é uma aplicação Web desenvolvida em AngularJS.
    Os arquivos de configuração do projeto da aplicação estão na pasta /gulp
    Os arquivos da aplicação ficam na pasta /src.
    A pasta src/release/ contém os arquivos gerados na build. O Gulp roda o comando no arquivo padrão gulpfile.js.
    O comando gulp build na pasta /guidelineApp executa a versão final da aplicação web da Documentação da Guideline RNP.

    Para utilizar a versão final deste projeto em algum webserver, copie a pasta /dist ou o seu conteúdo para um diretório raiz do seu projeto no WebServer.

  • /release

    Arquivos gerados pelo Gerador do Build

    Contém os arquivos na qual o gerador de build cria.

    Nota Você deve importar o arquivo vendors.css e o guideline.css na tag HEAD respectivamente, e o arquivo vendors.js e o arquivo guideline.js no final da tag Body e antes dos arquivos de projeto. Ao gerar o comando gulp build, a pasta raíz chamada /release e a pasta /guidelineApp/src/release serão atualizadas com os arquivos gerados pela build.

Criando um build

  • Abra uma instância do terminal/cmd e vá para a pasta onde você clonou o projeto RNP
  • Acesse a pasta /src.
  • Instale as dependências do node utilizando o comando sudo npm install.
  • Instale as dependências do projeto utilizando o comando sudo bower install.
  • Execute o comando gulp build se você estiver utilizando uma plataforma Unix ou se estiver na pasta do projeto no Prompt de Comando se você estiver no Windows para iniciar o build.

Sucesso! Verifique a pasta /release e /guidelineApp/src/release no diretório raiz do projeto.
Os arquivos release/guideline-VERSION.min.js e release/vendors-VERSION.min.js serão criados.

É gerado uma versão de download no formato .zip dentro da pasta /release, assim como está presente neste Site da Documentação.

A versão não minificada desses arquivos encontra-se na pasta /release. Eles são melhores para o desenvolvimento pois são apenas concatenados.

Atenção Certifique-se de sempre utilizar a versão minificada dos arquivos em produção. Esses arquivos já foram otimizados para obter a melhor performance de carregamento.

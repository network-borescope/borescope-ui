Versionamento

A guideline, conforme está desenvolvida, contempla com eficiência de funcionalidade e design/layout dos seguintes navegadores e versões:

Guideline X.Y.Z

Onde:

  • X, deve ser incrementado se qualquer alterações realizadas tornar a versão não mais compatível com a anterior. Patch e versão secundária devem ser alteradas para 0 quando a versão principal é incrementado.
  • Y, deve ser incrementado para novas funcionalidades, funcionalidades marcadas como obsoletas e melhorias mantendo a compatibilidade.
  • Z, deve ser incrementado apenas para correções de bugs, mantendo a compatibilidades da versão anterior.

Padrões de versionamento

  • Será usado padrão de versionamento na guideline conforme padrões internacionais.
  • Semantic Versioning: SemVer.
